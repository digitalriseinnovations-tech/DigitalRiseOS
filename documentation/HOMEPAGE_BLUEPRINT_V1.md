## 1. Purpose

This document defines the complete specification for the Digital Rise OS homepage. It serves as the authoritative reference for all design, engineering, and content decisions related to the homepage build.

This specification converts the Sprint A product brief into structured requirements that can be directly executed by the engineering and design team without additional interpretation or clarification.

All sections, interactions, animations, content structures, and acceptance criteria defined here are requirements, not suggestions.

---

## 2. Business Goal

**Primary conversion goal:** Generate qualified demo bookings.

Every element on the homepage exists to serve this goal. Visual design, copywriting, interaction design, section order, and CTAs are all subordinate to this single outcome.

**Supporting business goals (in priority order):**
1. Build trust with the target audience through authority signals, results, and social proof.
2. Explain the platform visually — minimize reliance on long-form copy.
3. Demonstrate industry expertise and platform breadth.
4. Capture warm leads through the AI Growth Audit lead magnet.
5. Drive secondary demo bookings from users not yet ready to commit.

**Brand positioning:**
The homepage must feel like a premium SaaS product — specifically in the class of Linear, Stripe, HubSpot, and Vercel. It must not read or look like an AI agency, a services company, or a generic automation tool.

---

## 3. User Goal

### Primary Audience

The homepage speaks to business owners and operators in the following verticals:

| Industry Vertical |
|---|
| Daycares |
| Medical Clinics |
| Dental Clinics |
| Contractors |
| Home Services |
| Law Firms |
| Accounting Firms |
| Local Retail |
| Restaurants |
| Fitness Studios |
| Professional Services |

### User State on Arrival

The typical homepage visitor arrives with one or more of the following problems:

- They are losing customers to missed calls or slow response times.
- They do not have a reliable follow-up system.
- They are managing too much manually.
- Their online reviews do not reflect the quality of their business.
- They want to grow but do not know where to start.

### What the User Needs to Feel by the End of the Page

1. "This platform understands my problem."
2. "I can see how this works."
3. "Other businesses like mine have gotten real results."
4. "I know exactly what to do next."

### Homepage Narrative Arc

The page tells a story in sequence. The user must feel the story progressing as they scroll:

```
Businesses lose customers.
        ↓
Digital Rise captures them.
        ↓
AI follows up automatically.
        ↓
More appointments are booked.
        ↓
More reviews are generated.
        ↓
Revenue grows.
```

This narrative is the through-line for all copy and section sequencing decisions. Never deviate from this order.

## 4. Homepage Architecture

### Page Structure Overview

The homepage consists of 12 sections, one persistent navigation, and three floating UI elements.

```
┌─────────────────────────────────┐
│  Navigation (Sticky)            │
├─────────────────────────────────┤
│  Section 01 — Hero              │
│  Section 02 — Trusted Results   │
│  Section 03 — Problem Statement │
│  Section 04 — Platform Preview  │
│  Section 05 — Platform Modules  │
│  Section 06 — Industry Selector │
│  Section 07 — Customer Journey  │
│  Section 08 — AI Growth Audit   │
│  Section 09 — Customer Success  │
│  Section 10 — Resources         │
│  Section 11 — FAQ               │
│  Section 12 — Final CTA         │
├─────────────────────────────────┤
│  Footer                         │
└─────────────────────────────────┘

Floating: Sticky CTA bar | WhatsApp | Dark Mode Toggle
```

### CTA Discipline

Every section contains exactly one primary CTA. There are no dead ends on this page. A user who reads to the end of any section always has a clear next action available.

## 5. Section-by-Section Specification

---

### Section 01 — Hero

**Purpose:** Make an immediate, undeniable first impression. Communicate the platform's value in under 5 seconds. Capture the user's intent to continue scrolling or take immediate action.

**Visual approach:** Bold statement headline. Minimal body text. Dominant product visual (dashboard or platform screenshot). Two CTAs.

**Content requirements:**

| Element | Requirement |
|---|---|
| Headline | Large. Maximum 8 words. Communicates the outcome, not the feature. |
| Subheading | Maximum 2 lines. Explains what Digital Rise OS does and for whom. |
| Primary CTA | "Book a Demo" or "Book Growth Audit" |
| Secondary CTA | "See How It Works" (scrolls to Section 04) |
| Hero visual | Dashboard or platform UI — not a stock photo, not a person at a desk |
| Trust signals | Optional: "Join X businesses" or logo strip immediately below CTAs |

**Layout:**
- Desktop: two-column — headline + CTAs left, product visual right
- Mobile: single column — headline first, visual below, CTAs last

**Visual style:**
- Background: white or very dark navy — not gray
- No decorative illustrations that compete with the product visual
- Generous whitespace

---

### Section 02 — Trusted Results

**Purpose:** Immediately validate the platform with quantified outcomes. No copy — only numbers and labels.

**Content requirements:**

| Metric | Label |
|---|---|
| 95% | Lead Response Rate |
| 40% | More Bookings |
| 24/7 | AI Employees Active |
| 100% | Lead Tracking |

**Visual approach:**
- Four stat blocks in a horizontal row (desktop), 2×2 grid (tablet), stacked (mobile)
- Numbers are the dominant typographic element — large, bold
- Labels are secondary — small, subdued

**Animation requirement:**
- Numbers count up from zero when the section enters the viewport
- Each counter animates independently with a staggered start (see Section 7: Animation Requirements)

**No CTA in this section.** This section is purely credibility-building, positioned immediately after the hero to reinforce the user's initial interest before they scroll further.

---

### Section 03 — Why Businesses Lose Customers

**Purpose:** Identify the user's pain directly. Make them feel understood before presenting the solution.

**Narrative role:** This is the "problem" section. It names the friction the target audience lives with daily. Done correctly, the user reads this and thinks "that is exactly my situation."

**Content requirements:**

Each problem is presented as a card with an icon and a short label. No paragraph copy.

| Problem | Label |
|---|---|
| Missed Calls | Calls go unanswered during busy hours |
| Slow Replies | Leads go cold before anyone responds |
| No Follow Up | Interested prospects are never contacted again |
| Poor Reviews | Happy customers never leave reviews |
| Manual Work | The team is doing work that should be automated |

**Visual approach:**
- Cards arranged in a grid — see `08 Cards.md` for feature card specification
- Each card: icon (from Lucide, or custom) + problem title + one-line description
- No heavy copy. No paragraphs.
- Subtle, illustrated icons preferred over generic line icons for this section

**CTA below section:** "See How We Fix This →" → scrolls to Section 04

---

### Section 04 — Meet Digital Rise OS

**Purpose:** Show the product in action. This is the platform preview moment — the section that converts curiosity into conviction.

**Content requirements:**
- A large, high-fidelity product visual (dashboard UI, analytics view, or AI conversation interface)
- Heading that names the platform: "Meet Digital Rise OS"
- Subheading: one line, outcome-focused
- Interactive elements within or around the visual (hover to reveal features, tab switching between views)
- Animated indicators highlighting key areas of the UI

**Visual approach:**
- This section may use a dark background to make the product UI stand out
- The UI must be modern, polished, and SaaS-grade — no placeholder content visible
- Hover effects reveal feature callouts positioned at points of interest on the UI
- Animation: elements of the UI appear in sequence as the section enters the viewport

**CTA:** "Get a Live Demo" (primary CTA)

---

### Section 05 — Platform Modules

**Purpose:** Communicate the breadth and capability of the platform. Each module is a distinct value driver. The user should understand the full scope of what they get.

**Modules to include:**

| Module Name | Role |
|---|---|
| Lead Capture | Captures visitors before they leave |
| AI Employee | Responds to leads instantly, 24/7 |
| CRM | Organizes and tracks every contact |
| Reviews | Generates and manages online reviews |
| Automation | Eliminates repetitive manual tasks |
| Analytics | Tracks growth and performance in real time |
| Growth Audit | Identifies exactly where to improve |

**Visual approach:**
- Grid layout — see `05 Grid System.md` for column specifications
- Each module: icon container + module name + one-line description
- Consistent card style across all modules — no hierarchy between modules in this section

**CTA below section:** "Explore the Platform →" or "Book a Demo"

---

### Section 06 — Industry Selector

**Purpose:** Personalize the experience. Show the user that this platform is built specifically for their type of business.

**Interaction model:**
- The user sees a list or grid of industry names
- Selecting an industry updates the content in the section — the use case, the metrics, and the visual shift to reflect that specific industry
- No page reload. Instant update via JavaScript state.

**Industries to include:**

Daycares, Medical Clinics, Dental Clinics, Contractors, Home Services, Law Firms, Accounting Firms, Local Retail, Restaurants, Fitness Studios, Professional Services

**Per-industry content required:**

Each industry selection must update:
1. A headline specific to that industry (e.g., "Built for Dental Clinics")
2. A short description of how Digital Rise OS solves that industry's specific problem
3. A relevant metric or result (e.g., "Dental clinics see 40% more booked appointments")
4. The product visual or illustration if applicable

**Default state:** First industry in the list is pre-selected on load.

**Transition animation:** Content within the section cross-fades when the industry changes. Duration 200–280ms.

**CTA within section:** "See How It Works for [Selected Industry]" — updates with the selection.

---

### Section 07 — Customer Journey

**Purpose:** Show the user exactly how the platform transforms a visitor into a loyal customer. Make the process feel simple and inevitable.

**Journey stages:**

```
Visitor → Lead → AI → Booking → Customer → Review → Repeat Business
```

**Visual approach:**
- Horizontal flow on desktop (stages connected by animated arrows or a progress line)
- Vertical stack on mobile
- Each stage: icon + label + one-line description
- The flow animates in sequence as the section enters the viewport (each stage appears after the previous, staggered by 100ms)
- Connecting arrows or lines draw progressively between stages

**Content per stage:**

| Stage | Label | Description |
|---|---|---|
| Visitor | Visitor Arrives | Someone finds your business online |
| Lead | Lead Captured | AI captures their contact instantly |
| AI | AI Responds | Immediate, personalized follow-up |
| Booking | Booking Made | Appointment secured automatically |
| Customer | Customer Converted | Visitor becomes a paying customer |
| Review | Review Generated | Happy customer leaves a 5-star review |
| Repeat Business | Cycle Continues | Automated re-engagement brings them back |

**CTA below section:** "See the Full Platform" or "Book a Demo"

---

### Section 08 — AI Growth Audit

**Purpose:** Generate qualified leads. This is the primary lead magnet. The Growth Audit offers the user direct, personalized value in exchange for their contact information and intent signal.

**Visual approach:**
- This is the largest, most visually prominent CTA section on the page
- Background: dark (navy or brand gradient) to create maximum visual contrast after several light sections
- Heading: bold, large, outcome-focused — "Find Out Exactly How Much Revenue You Are Leaving on the Table"
- Subheading: 1–2 lines explaining what the audit provides
- CTA button: high contrast, large (`lg` size), centered

**What the audit offers (supporting copy):**
- Identify your top 3 growth gaps
- See exactly where leads are being lost
- Get a custom action plan — free

**Form or CTA:**
If this section uses an inline form:
- Fields: Name, Business Name, Industry (dropdown), Email, Phone
- Submit button: "Get My Free Growth Audit"
- Below form: trust line — "No credit card. No obligation. Results in 24 hours."

If this section links to a separate audit page:
- CTA button navigates to `/growth-audit`
- Button label: "Start My Free Audit"

**This section requires the highest visual investment on the page.**

---

### Section 09 — Customer Success

**Purpose:** Provide social proof through real customer voices. Build trust by showing that businesses like the visitor's have already succeeded with Digital Rise OS.

**Visual approach:**
- Interactive testimonial wall — not a carousel
- The wall displays multiple testimonials simultaneously, visible without interaction
- Optionally: filterable by industry (consistent with Section 06 industry selector)
- Uses the testimonial card specification from `15 Testimonials.md`

**Per testimonial required data:**
- Full name
- Business name and industry
- Star rating (displayed as 5-star graphic)
- Quote text (40–100 words)
- Real photo avatar

**Wall layout:**
- Masonry or 3-column grid — see `08 Cards.md`
- On mobile: single column, scrollable

**No CTA inside individual testimonials.** One CTA below the section: "Join Them — Book a Demo"

---

### Section 10 — Resources

**Purpose:** Establish thought leadership and provide continued value for users who are not yet ready to book a demo. The resources section captures mid-funnel interest.

**Visual approach:**
- "Magazine style" — bold, editorial card layout
- Featured resource: large card spanning full width or 8 columns
- Supporting resources: 2–3 smaller cards below

**Content types to include:**
- Blog articles
- Case studies
- Guides / playbooks
- Platform feature deep-dives

**Card requirements:** See `08 Cards.md — Blog/Article Card`

**CTA:** "Read More" on individual cards. Section-level CTA: "Visit the Resource Centre →"

---

### Section 11 — FAQ

**Purpose:** Remove final objections and answer common questions before the user reaches the final CTA. A well-answered question converts a hesitant visitor.

**Visual approach:**
- Accordion pattern — see `16 FAQ.md`
- Minimal. One column. Clean.
- Maximum 8 questions

**Suggested question categories:**
- What the platform does
- How it works
- Pricing and commitment
- Setup and onboarding
- Industry fit
- AI capabilities

**CTA below section:** "Have more questions? [Talk to us →]" — links to contact page or opens a chat.

---

### Section 12 — Final CTA

**Purpose:** The last and clearest conversion opportunity on the page. Every user who reaches this section has consumed the full homepage story. The final CTA must feel inevitable.

**Two-option model:**
The final CTA section presents two paths — one higher commitment, one lower.

| Option | Label | Action |
|---|---|---|
| Primary | "Book Your Growth Audit" | Opens audit form or navigates to audit page |
| Secondary | "Schedule a Demo" | Opens calendar booking link |

**Visual approach:**
- Full-width section, dark background (navy or brand gradient)
- Heading: maximum 10 words, outcome-focused
- Subheading: 1 line, reducing friction ("Free. No commitment. 20 minutes.")
- Two buttons, centered, side by side on desktop, stacked on mobile
- No other content competing for attention in this section

## 6. Interaction Requirements

### Navigation

- Sticky top bar that remains visible throughout the entire page scroll.
- On scroll: navigation adds a subtle background and shadow — transitions from transparent to `--color-neutral-0` with `backdrop-filter: blur(8px)` over 200ms.
- "Book Demo" button in the navigation is always visible.
- On mobile: hamburger menu opens a full-width drawer.
- Smooth scroll to target section when anchor navigation links are clicked.

### Industry Selector (Section 06)

- Click or tap on an industry name/card to select it.
- Selected state: visual indicator (highlighted border or background) on the selected industry.
- Content within the section updates instantly without page reload.
- Cross-fade transition on content change: opacity 0 → 1, 200ms ease-out.
- Default selection: first industry on load.

### Platform Preview (Section 04)

- Desktop: hovering over defined hotspots on the product visual reveals a callout label describing that feature.
- Callout appears on hover: fade in, 150ms ease-out.
- Callout dismisses when hover ends: fade out, 100ms ease-in.
- Tab switching: if the preview has multiple views (e.g., CRM, Analytics, AI Chat), tabs allow switching between them. Active tab content cross-fades in 200ms.

### Testimonial Wall (Section 09)

- All testimonial cards are visible simultaneously — no carousel, no hidden slides.
- Optional: filter buttons above the wall allow filtering by industry.
- Filter click: non-matching cards fade out (opacity → 0.2) and matching cards remain at full opacity. Transition: 200ms.
- No infinite scroll. Display a curated set.

### FAQ Accordion (Section 11)

- Click or tap a question to expand the answer.
- Only one question open at a time. Opening a new question closes the currently open one.
- Expand animation: height 0 → auto + opacity 0 → 1, 250ms ease-out.
- Collapse animation: height → 0 + opacity → 0, 180ms ease-in.
- Chevron icon rotates 180° when expanded.

### Floating Elements

**Sticky CTA Bar:**
- Appears after the user scrolls past the hero section (past 100vh).
- Contains: "Book a Demo" button + optional secondary action.
- Positioned: fixed, bottom of viewport, full width or right-aligned.
- Entry animation: slide up from below viewport, 280ms ease-out.
- Does not appear until hero CTA has scrolled out of view.

**WhatsApp Button:**
- Fixed position, bottom-right corner, above the sticky CTA bar.
- 56px × 56px circular button, WhatsApp green, WhatsApp icon.
- Tooltip "Chat with us" appears on hover.
- Opens WhatsApp web link in new tab.

**Dark Mode Toggle:**
- Accessible via the sticky floating elements or embedded in the navigation.
- Toggles between light and dark color themes.
- Theme preference persisted to `localStorage`.
- Toggle animation: sun/moon icon transition, 200ms.

## 7. Animation Requirements

All animations conform to the motion system defined in `12 Motion System.md`. The following specifications are homepage-specific.

### Global Animation Rules

- All scroll-triggered animations activate when the element enters 10% of the viewport.
- All scroll-triggered animations play once. They do not replay on scroll-back.
- All animations use `transform` and `opacity` only — never `width`, `height`, `top`, `left`, or `margin`.
- All animations respect `prefers-reduced-motion: reduce` — when active, elements appear immediately without animation.

### Hero Section

| Element | Animation | Duration | Easing |
|---|---|---|---|
| Headline | Fade in + translateY(20px → 0) | 500ms | ease-out |
| Subheading | Fade in + translateY(16px → 0), delayed 100ms | 450ms | ease-out |
| CTA buttons | Fade in + translateY(12px → 0), delayed 200ms | 400ms | ease-out |
| Hero visual | Fade in + scale(0.96 → 1), delayed 150ms | 600ms | ease-out |

### Stat Counters (Section 02)

| Property | Value |
|---|---|
| Trigger | Section enters viewport |
| Count direction | 0 → final value |
| Duration per counter | 1200ms |
| Easing | ease-out (decelerates near final value) |
| Stagger | 120ms between each counter |

### Scroll Entrance (Default — All Other Sections)

| Property | Value |
|---|---|
| Default entry | opacity 0 → 1 + translateY(20px → 0) |
| Duration | 350ms |
| Easing | ease-out |
| Stagger (for lists/grids) | 60ms per item, starting after 60ms base delay |

### Customer Journey Flow (Section 07)

| Property | Value |
|---|---|
| Animation type | Each stage fades + slides in sequentially |
| Connecting line/arrow | Draws from left to right progressively |
| Stagger between stages | 100ms |
| Duration per stage | 300ms ease-out |

### Magnetic Buttons

- Hero CTA buttons and the final CTA buttons implement a magnetic hover effect.
- On mouse move within 80px of the button: the button shifts slightly toward the cursor (transform: translate) — maximum offset 8px in any direction.
- On mouse leave: button returns to resting position over 300ms ease-out.
- Applies to desktop only. Not applied on touch devices.

### Parallax

- Applies to hero section background and select decorative background elements only.
- Parallax rate: background moves at 30–50% of scroll speed relative to foreground content.
- Does not apply to text or interactive elements.
- Disabled when `prefers-reduced-motion: reduce` is active.

### Smooth Scrolling

- Native CSS `scroll-behavior: smooth` applied globally.
- Anchor link navigation uses JavaScript-controlled smooth scroll for fine control.
- Scroll duration to target: 600–900ms depending on distance, ease-in-out.
